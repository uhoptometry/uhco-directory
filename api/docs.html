Introduction

The UHCO Directory API provides programmatic read access to the University of Houston College of Optometry's people, organizations, and classification data. It is a JSON REST API served under:

https://<host>/api/v1/

All responses are JSON with UTF-8 encoding. All endpoints are read-only (GET). Attempting any other HTTP method returns 405 Method Not Allowed.

Base URL
http://127.0.0.1/api/v1/
Versioning

The current version is v1, reflected in the URL path. Breaking changes will increment the version number.

Authentication

All endpoints (except the root discovery endpoint) require a valid API token.

Supply the token using either of these methods:

Authorization header (preferred)
Authorization: Bearer uhcs_<your-token>
Query string (for browser testing)
GET /api/v1/people?token=uhcs_<your-token>
Query string tokens are visible in server logs. Use the Authorization header in production clients.
Token scopes
ScopeAccess
readAll GET endpoints
writeReserved for future use
Secrets (Protected Data)

Certain records require both a valid token and a valid secret. Without a secret, those records are silently excluded from all results — the response will still succeed but will not include the protected people..

Protected by default

The following are excluded from every response unless a secret that unlocks them is supplied:

Protected dataUnlocked by secret granting…
Users with the Current-Student flagCurrent-Student
Users with the Alumni flagAlumni
Users in Academic Programs orgs
(OD Program, MS Program, PhD Program, Residency Program) 		Current-Student

Supply the secret alongside your token using either:

X-API-Secret header (preferred)
Authorization: Bearer uhcs_<your-token>
X-API-Secret: uhcs_sec_<your-secret>
Query string
GET /api/v1/people?token=uhcs_...&secret=uhcs_sec_...
If you filter by a protected flag or org (e.g. ?flag=Current-Student or ?org=OD+Program) without a valid secret, the response will return total: 0 and an empty data array rather than an error.

Errors

Errors return a JSON body with an error key and an appropriate HTTP status code.

{ "ERROR": "User not found" }
StatusMeaning
400Bad request — invalid parameter (e.g. non-numeric ID)
401Missing, malformed, expired, or invalid token
403Token lacks required scope or request IP is not allowed
404Resource not found
405HTTP method not allowed
500Internal server error

Endpoints

GET /api/v1/people

Returns a paginated list of directory people. Supports full-text search across name and email fields.

Query Parameters
ParameterTypeDefaultDescription
searchstringnoneSearches first name, last name, preferred name, and primary email.
flagstringnoneFilter to users with this flag (exact flag name, e.g. Staff, Faculty-Fulltime). Protected flags require a secret.
orgstringnoneFilter to users in this organization (exact org name, e.g. Clinical Operations).
limitinteger50Max records to return. Maximum allowed: 500.
offsetinteger0Number of records to skip (for pagination).
returnstringjsonResponse format. Accepted values: json, xml.
secretstringnoneAPI secret to unlock protected records (alternative to X-API-Secret header).
tokenstringnoneAPI token (alternative to Authorization header).
Response
{
  "TOTAL": 3319,
  "LIMIT": 50,
  "OFFSET": 0,
  "DATA": [ { "USERID": 1, "FIRSTNAME": "Jane", "LASTNAME": "Doe", ... } ]
}
Example Requests
GET /api/v1/people?token=uhcs_...
GET /api/v1/people?search=smith&limit=25&offset=50&token=uhcs_...
GET /api/v1/people?flag=Staff&token=uhcs_...
GET /api/v1/people?org=Clinical+Operations&token=uhcs_...
GET /api/v1/people?flag=Current-Student&secret=uhcs_sec_...&token=uhcs_...
GET /api/v1/people?flag=Alumni&secret=uhcs_sec_...&token=uhcs_...
GET /api/v1/people?org=OD+Program&secret=uhcs_sec_...&token=uhcs_...
GET /api/v1/people?return=xml&token=uhcs_...

See the Examples page for working code samples in JavaScript, Python, Node.js, ColdFusion, and curl.

GET /api/v1/quickpulls/gradclass

Returns active alumni for a single graduation year as a flat list, filtered to a specific academic program organization. Each result includes the matched PROGRAM, a combined FULLNAME, and image URLs for the interactive_roster, KIOSK_ROSTER, and KIOSK_PROFILE variants. This endpoint is protected because it exposes alumni data.

Authentication

Requires both a valid API token and a valid secret that unlocks the Alumni flag.

Query Parameters
ParameterTypeDefaultDescription
yearintegerrequiredGraduation year to filter by, for example 2026.
programstringrequiredAcademic program organization name. Allowed values: OD Program, PhD Program, MS Program.
secretstringnoneAPI secret with Alumni access, as an alternative to the X-API-Secret header.
tokenstringnoneAPI token, as an alternative to the Authorization header.
Response
{
    "TOTAL": 87,
    "DATA": [
        {
            "USERID": 314,
            "FIRSTNAME": "Alex",
            "MIDDLENAME": "",
            "LASTNAME": "Smith",
            "CURRENTGRADYEAR": 2026,
            "PROGRAM": "OD Program",
            "FULLNAME": "Alex Smith",
            "INTERACTIVEUSERIMAGE": "/published_images/interactive_roster/314.jpg",
            "KIOSKROSTERIMAGE": "/published_images/kiosk_roster/314.jpg",
            "KIOSKPROFILEIMAGE": "/published_images/kiosk_profile/314.jpg"
        }
    ]
}
Notes

Returned users must have the Alumni flag and belong to the selected academic program org. If a user does not have one or more of those image variants, the corresponding image field is still included with an empty string value.

Example Requests
GET /api/v1/quickpulls/gradclass?year=2026&program=OD+Program&secret=uhcs_sec_...&token=uhcs_...
GET /api/v1/quickpulls/gradclass?year=2026&program=PhD+Program&secret=uhcs_sec_...&token=uhcs_...
GET /api/v1/quickpulls/graduate

Returns a single alumni-oriented quickpull record by UserID, including academic fields, externship and hometown details, earned degrees, awards, and image URLs when available.

Authentication

Requires both a valid API token and a valid secret that unlocks the Alumni flag.

Query Parameters
ParameterTypeDefaultDescription
idintegerrequiredThe numeric UserID for the graduate record to fetch.
secretstringnoneAPI secret with Alumni access, as an alternative to the X-API-Secret header.
tokenstringnoneAPI token, as an alternative to the Authorization header.
Response
{
    "USERID": 314,
    "DATA": {
        "USERID": 314,
        "FIRSTNAME": "Alex",
        "MIDDLENAME": "",
        "LASTNAME": "Smith",
        "CURRENTGRADYEAR": 2026,
        "FIRSTEXTERNSHIP": "Houston Eye Associates",
        "SECONDEXTERNSHIP": "VA Eye Clinic",
        "HOMETOWNCITY": "Houston",
        "HOMETOWNSTATE": "TX",
        "DEGREES": [
            { "USERID": 314, "DEGREENAME": "O.D.", "UNIVERSITY": "University of Houston", "DEGREEYEAR": "2026" }
        ],
        "AWARDS": [
            { "USERID": 314, "AWARDNAME": "Dean's List", "AWARDTYPE": "Honor" }
        ],
        "INTERACTIVEUSERIMAGE": "/published_images/interactive_roster/314.jpg",
        "KIOSKROSTERIMAGE": "/published_images/kiosk_roster/314.jpg",
        "KIOSKPROFILEIMAGE": "/published_images/kiosk_profile/314.jpg"
    }
}
Notes

If the UserID does not exist, is inactive, or is not flagged as Alumni, the endpoint returns an empty object in DATA rather than a full profile. If any image variant is unavailable, its field is still present with an empty string value.

Example Requests
GET /api/v1/quickpulls/graduate?id=314&secret=uhcs_sec_...&token=uhcs_...
